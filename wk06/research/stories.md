Story S1: Filter Persistence
Situation: When I'm filtering tasks by project or deadline in a multi-tab workflow
Motivation: I want the filter selection to persist across tab switches and page reloads
Outcome: So I can maintain focus without constantly re-selecting filters
Underlying need: Because losing filter state causes cognitive overload and wastes time on repetitive actions

Evidence: Participant A (notes Q1 response), Participant C (notes Q3 response)
Inclusion risk: Cognitive, ADHD, Memory impairment
Type: Job story
WCAG: 3.2.2 On Input (AA) - Consistent behavior across sessions

Story S2: Completion Feedback & History
Situation: When I complete a task in my task manager
Motivation: I want to see visual confirmation and access a history of completed tasks
Outcome: So I can track my progress and feel motivated by accomplishments
Underlying need: Because disappearing tasks without feedback reduces motivation and makes it hard to review daily productivity

Evidence: Participant A (notes Q2 response), Participant C (notes Q2 response)
Inclusion risk: Cognitive, Motivation challenges, ADHD
Type: Job story
WCAG: 4.1.3 Status Messages (AA)

Story S3: Keyboard Accessibility for Motor Impairment
Situation: When I'm experiencing wrist pain or RSI and cannot use a mouse
Motivation: I want to access all task management functions using only keyboard navigation
Outcome: So I can continue working without pain or exclusion
Underlying need: Because reliance on pointing devices creates barriers for people with temporary or permanent motor impairments

Evidence: Participant A (notes Q3 response), Participant C (notes Q3 response)
Inclusion risk: Motor impairment, RSI, Temporary disability
Type: Job story
WCAG: 2.1.1 Keyboard (A), 2.1.3 Keyboard (No Exception, AAA)

Story S4: High Contrast for Environmental Accessibility
Situation: When I'm working outdoors in bright sunlight or have low vision
Motivation: I want text and interface elements to have sufficient contrast against backgrounds
Outcome: So I can read task information without straining or needing to change location
Underlying need: Because low contrast creates situational disability in bright environments and permanent exclusion for low vision users

Evidence: Participant B (notes Q2 response)
Inclusion risk: Low vision, Color-blindness, Situational (bright light)
Type: Job story
WCAG: 1.4.3 Contrast (Minimum, AA) - 4.5:1 for normal text

Story S5: Helpful Error Messages
Situation: When I make an input error while adding or editing a task
Motivation: I want clear, specific error messages that explain what went wrong and how to fix it
Outcome: So I can correct my mistake efficiently without frustration
Underlying need: Because generic "error" messages cause confusion, wasted time, and may lead to task duplication

Evidence: Participant B (notes Q3 response)
Inclusion risk: Cognitive, Low digital literacy, Non-native speakers
Type: Job story
WCAG: 3.3.1 Error Identification (A), 3.3.3 Error Suggestion (AA)

Story S6: Priority Management for Long Lists
Situation: When I'm managing 20+ tasks and need to identify what's most important
Motivation: I want a way to pin or highlight critical tasks to keep them visible
Outcome: So I don't miss important deadlines or commitments
Underlying need: Because long, undifferentiated lists cause cognitive overload and increase the risk of missing critical items

Evidence: Participant C (notes Q1 response)
Inclusion risk: Cognitive, ADHD, Anxiety
Type: Job story
